============================================================
1) Mini Database Engine (gen SQLite Lite)
============================================================

Un proiect excelent pentru CV, pentru că arată:
    • structuri de date avansate
    • gestionarea memoriei
    • parsing
    • optimizări
    • graphics programming (RayLib GUI)

============================================================
PROGRES
============================================================

[✓] IMPLEMENTAT:
    • AST complet
        - Statement (baza abstracta)
        - SelectStatement
        - InsertStatement
        - DeleteStatement
        - CreateStatement
        - Condition
        - Columns
    • Lexer
        - TokenType, Token (fisiere separate)
        - Tokenizare completa: keywords, identifiers,
          numbers, strings, operators, punctuation, wildcard
    • Parser
        - parseSelect, parseInsert, parseDelete, parseCreate
        - parseCondition
        - validare query invalid
    • Storage
        - PageHeader
        - Page (addRow, getRows, hasSpace, getRowCount)
        - PageManager (readPage, writePage, insertRow, getAllRows)
    • Catalog
        - createTable, tableExists, getColumns
    • Table
        - leaga Catalog + PageManager
        - API simplu pentru operatii pe tabele (insert/select/delete)
    • Engine
        - executa Statements peste Table/PageManager
        - leaga parser-ul de storage (end-to-end DB functional)
    • B+ Tree Index
        - IndexRecord (key, pageId, offset)
        - BPlusNode (isLeaf, keys, children, records, next, parent)
        - BPlusTree (insert, search, clear, splitLeaf, splitInternal)
        - integrat in Table: cautare O(log n) pe prima coloana INT
        - rebuild automat al indexului la crearea obiectului Table
        - reset + rebuild la DELETE
    • Page Cache LRU
        - CacheEntry (page, isDirty)
        - LRUCache (get, put, evict, clear, flushAll)
        - std::list pentru ordinea LRU + std::unordered_map pentru O(1) lookup
        - integrat in PageManager: readPage verifica cache inainte de disc
        - writePage scrie in cache cu isDirty=true, nu direct pe disc
        - flushAll() in destructorul PageManager garanteaza persistenta
        - clear() folosit in clearAll() pentru reset fara scriere pe disc

[✓] Write-Ahead Logging (WAL)
        - WALEntry (type, table, rowData, committed)
        - WALManager (logInsert, commit, recover, readLog)
        - integrat in Engine: logInsert inainte de INSERT, commit dupa
        - recover() apelat automat in constructorul Engine la pornire
        - testat: date recuperate corect dupa crash simulat

[✗] LIPSESTE:
    • RayLib GUI

============================================================
ARHITECTURA
============================================================

1. SQL Parser
   Primeste un string gen SELECT * FROM users WHERE id = 5
   si il transforma intr-un AST pe care engine-ul il executa.

2. Page-Based Storage
   Fisierul mydb.db este impartit in pagini de 4KB.
   Randurile sunt ambalate in pagini — imita comportamentul
   real al HDD/SSD (citesti/scrii blocuri, nu octeti).

3. B+ Tree Index
   Structura arborescenta pentru cautare rapida dupa cheie
   primara. Fara index: scan complet. Cu B+ Tree: O(log n).

4. Page Cache (Buffer Pool)
   Tine in RAM paginile recent accesate (politica LRU).
   Cache hit -> RAM. Cache miss -> disc + adauga in cache.

5. Write-Ahead Log (WAL)
   Orice modificare e scrisa intai in log, apoi pe disc.
   La crash -> recovery din log. Fundamentul durabilitatii.

RAYLIB GUI
============================================================

  Layout:
  ┌──────────────────────────────────────────────────┐
  │                Mini DB Engine                    │
  ├──────────────────────────────────────────────────┤
  │  ┌────────────────────────────────┐  ┌───────┐  │
  │  │ SELECT * FROM users            │  │  RUN  │  │
  │  └────────────────────────────────┘  └───────┘  │
  ├──────────────────────────────────────────────────┤
  │   id  │  name  │  age                            │
  │  ─────┼────────┼──────                           │
  │   1   │  Ion   │  25    <- citit din mydb.db     │
  │   2   │  Ana   │  30                             │
  ├──────────────────────────────────────────────────┤
  │  [LOG] ✅ Query executed in 0.002ms – 2 rows     │
  │  [LOG] ❌ ERROR: Table 'x' does not exist        │
  └──────────────────────────────────────────────────┘

  Componente:
    • SQL Input field   - text editabil, Enter sau buton RUN
    • Buton RUN         - executa query-ul curent
    • Results table     - grid dinamic bazat pe schema tabelei
                          date citite LIVE din mydb.db
    • Log panel         - timestamp + mesaje
                          verde = succes | rosu = eroare

  Flow:
    SQL Input -> Parser -> Engine -> PageManager
                                         |
                                    mydb.db (disc)
                                         |
                               rows deserializate -> UI
