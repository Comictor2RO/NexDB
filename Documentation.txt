================================================================
DOCUMENTATIE — Mini Database Engine
Ce face fiecare clasa si metodele ei
================================================================

----------------------------------------------------------------
AST/ — Abstract Syntax Tree
----------------------------------------------------------------

Statement (clasa de baza abstracta)
    Rol: Interfata comuna pentru toate tipurile de comenzi SQL.
    Metode:
        execute() — virtual pure; fiecare tip de statement o implementeaza.

SelectStatement
    Rol: Reprezinta comanda SELECT (coloane, tabela, conditie WHERE optionala).
    Metode:
        getTable(), getColumns(), getCondition() — return valorile stocate.
        setTable(), setColumns(), setCondition() — seteri.
        execute() — override; logica reala e in Engine.

InsertStatement
    Rol: Reprezinta comanda INSERT (tabela, coloane optionale, valori).
    Metode:
        getTable(), getValues(), getColumns() — geteri.
        setTable(), setValues(), setColumns() — seteri.
        execute() — override.

DeleteStatement
    Rol: Reprezinta comanda DELETE (tabela, conditie WHERE optionala).
    Metode:
        getTable(), getCondition() — geteri.
        setTable(), setCondition() — seteri.
        execute() — override.

CreateStatement
    Rol: Reprezinta comanda CREATE TABLE (nume tabela, lista coloane cu tipuri).
    Metode:
        getTable(), getColumns() — geteri.
        setTable(), setColumns() — seteri.
        execute() — override.

Condition (struct)
    Rol: O conditie WHERE: coloana, operator, valoare.
    Campuri: column, op, value.

Columns (struct)
    Rol: O coloana din schema: nume si tip.
    Campuri: name, type.

Row (struct)
    Rol: Un rand de date ca lista de string-uri (valorile pe coloane).
    Campuri: values (vector<string>).

----------------------------------------------------------------
Frontend/ — Lexer si Parser
----------------------------------------------------------------

TokenType (enum)
    Rol: Tipurile de token-uri emise de Lexer.
    Valori: KEYWORD, IDENTIFIER, NUMBER, STRING, OPERATOR, PUNCTUATION, WILDCARD, END_OF_FILE.

Token (struct)
    Rol: Un token (tip + valoare string).
    Campuri: type, value.

Lexer
    Rol: Transforma un string SQL intr-un vector de Token-uri.
    Metode:
        Lexer(input) — constructor.
        tokenize() — parcurge string-ul, apeleaza readWord/readNumber/readString/etc., returneaza vector<Token>.

Parser
    Rol: Transforma vector<Token> intr-un Statement* (AST).
    Metode:
        Parser(tokens) — constructor.
        parse() — citeste primul keyword si apeleaza parseSelect/parseInsert/parseDelete/parseCreate; returneaza nullptr daca query invalid.
    Metode private: parseSelect(), parseInsert(), parseDelete(), parseCreate(), parseCondition(), currentToken(), consumeToken().

----------------------------------------------------------------
Storage/ — Pagini si cache
----------------------------------------------------------------

PageHeader (struct)
    Rol: Header fix 12 bytes la inceputul fiecarei pagini.
    Campuri: pageId, freeSpace, rowNumber.

Page
    Rol: O pagina de 4KB in memorie; stocheaza header + randuri (separate prin \n).
    Metode:
        Page(), Page(pageId) — constructori.
        addRow(row) — adauga un rand, actualizeaza header; returneaza false daca nu e loc.
        getRows() — citeste din buffer si returneaza vector de string-uri (randurile).
        hasSpace(rowSize) — verifica daca mai e loc.
        getPageId(), getFreeSpace(), getRowCount() — geteri din header.
        getBuffer() — returneaza char* la data[] pentru I/O.

PageManager
    Rol: Gestioneaza fisierul .db: citeste/scrie pagini, insereaza randuri, intoarce toate randurile.
    Metode:
        PageManager(filename) — deschide/creaza fisierul binar, calculeaza numberOfPages.
        insertRow(row) — pune randul pe ultima pagina cu loc sau creeaza pagina noua; returneaza true/false.
        getAllRows() — citeste toate paginile (prin cache) si concateneaza randurile.
        clearAll() — goleste fisierul si cache-ul.
    Private: readPage(pageId), writePage(pageId, page) — I/O pe disc; folosesc LRUCache.

CacheEntry (struct)
    Rol: Un slot din cache: pagina + flag isDirty.
    Campuri: page, isDirty.

LRUCache
    Rol: Cache LRU pentru pagini: get/put O(1), evict la cea mai veche folosire.
    Metode:
        LRUCache(capacity, file) — constructor.
        get(pageId) — returneaza pointer la pagina din cache sau nullptr; muta la cap (cel mai recent).
        put(pageId, page, isDirty) — pune/actualizeaza pagina la cap; daca e plin apeleaza evict().
        clear() — goleste lista si map-ul (fara scriere pe disc).
        flushAll() — evict la toate paginile (scrie pe disc cele dirty).
    Private: evict() — scoate ultima pagina; daca e dirty o scrie pe disc; o sterge din cache.

----------------------------------------------------------------
Catalog/
----------------------------------------------------------------

Catalog
    Rol: Tine schema bazei: map nume_tabela -> lista de coloane (Columns).
    Persistență: schema se salvează în catalog.dat (același folder cu .exe / cwd).
    La pornire: load() citește catalog.dat; se adaugă în catalog doar tabelele
    al căror fișier .db există (std::filesystem::exists(nume + ".db")). La final
    load() apelează save(), astfel catalog.dat rămâne sincronizat cu discul:
    dacă ai șters emp.db, la repornire emp nu mai e încărcat și e șters și din
    catalog.dat, deci SELECT * FROM emp dă "table does not exist" (nu se recreează emp.db).

    Metode:
        Catalog() — constructor; apelează load().
        createTable(name, columns) — adauga tabela daca nu exista; apelează save().
        tableExists(name) — true/false.
        getColumns(name) — returneaza vector<Columns>; gol daca tabela nu exista.
    Private:
        CATALOG_FILE — "catalog.dat".
        load() — deschide catalog.dat; pentru fiecare linie (format: nume|col1 tip1|col2 tip2|...),
                 parsează și adaugă în columns doar dacă există nume.db; la final apelează save().
        save() — rescrie catalog.dat cu toate tabelele din columns.

----------------------------------------------------------------
Table/
----------------------------------------------------------------

Table
    Rol: Leaga Catalog de PageManager; serializare/deserializare randuri; index B+ Tree pe prima coloana INT.
    Metode:
        Table(name, scheme) — deschide name.db, rebuild index din date existente.
        insertRow(row) — serializeaza row la "v1|v2|...", trimite la PageManager, actualizeaza index (key = prima coloana INT).
        selectRow(cond) — daca cond e WHERE pe prima coloana INT cu =, foloseste index.search(); altfel full scan; filtreaza cu evaluateCondition.
        deleteRow(cond) — citeste toate randurile, pastreaza cele care nu se potrivesc, clearAll(), index.clear(), rescrie randurile.
    Private: rebuildIndex() — citeste toate randurile de pe disc si reumple B+ Tree.
    Functie: evaluateCondition(cond, row, schema) — compara valoarea din row la coloana cond.column cu cond.value folosind cond.op.

----------------------------------------------------------------
Indexing/ — B+ Tree
----------------------------------------------------------------

IndexRecord (struct)
    Rol: O intrare din index: cheie + pozitie (pageId, offset = index global al randului).
    Campuri: key, pageId, offset.

BPlusNode (struct)
    Rol: Un nod din B+ Tree (intern sau frunza).
    Campuri: keys, children (pentru nod intern), records (pentru frunza), next (urmatoarea frunza), parent, isLeaf.

BPlusTree
    Rol: Index B+ Tree pentru cautare dupa cheie (O(log n)).
    Metode:
        BPlusTree(order) — constructor; order = nr max chei per nod.
        insert(key, pageId, offset) — gaseste frunza, insereaza; la overflow apeleaza splitLeaf/splitInternal.
        search(key) — returneaza IndexRecord* la cheia data sau nullptr.
        clear() — elibereaza toate nodurile, root = nullptr.
    Private: findLeaf(key), splitLeaf(), splitInternal(), deleteNode().

----------------------------------------------------------------
WALManager/
----------------------------------------------------------------

WALEntry (struct)
    Rol: O intrare din log: tip operatie, tabela, date rand, flag committed.
    Campuri: type, table, rowData, committed.

WALManager
    Rol: Write-Ahead Log: scrie operatii inainte de aplicarea pe .db; la pornire recover() reaplica necommitted.
    Metode:
        WALManager(filename) — deschide/creaza fisierul .wal (append).
        logInsert(table, rowData) — scrie linie "INSERT|table|rowData|0" si flush.
        commit() — citeste log-ul, marcheaza ultima intrare committed=1, rescrie fisierul.
        recover(engine) — citeste log-ul, pentru fiecare intrare necommitted de tip INSERT executa query prin engine si commit.
    Private: readLog() — citeste toate liniile din .wal si returneaza vector<WALEntry>.

----------------------------------------------------------------
Engine/
----------------------------------------------------------------

Engine
    Rol: Punct central: primeste SQL (sau Statement*), executa pe Catalog + Table; pentru INSERT scrie in WAL inainte.
    Metode:
        Engine(catalog) — constructor; deschide engine.wal, apeleaza wal.recover(*this).
        execute(statement) — dynamic_cast la tip; apeleaza executeCreate/executeInsert/executeDelete/executeSelect.
        query(sql) — Lexer + Parser + parse(); daca e SELECT verifica tableExists, construieste Table, selectRow(), returneaza vector<Row>; altfel verifica tableExists pentru INSERT/DELETE, apeleaza execute(stmt); arunca daca query invalid sau tabela inexistenta.
    Private: executeCreate(), executeInsert(), executeDelete(), executeSelect().

----------------------------------------------------------------
GUI/
----------------------------------------------------------------

GUI
    Rol: Interfata RayLib: fereastra 1280x720, input SQL, grid rezultate, log; executie la Enter sau buton RUN.
    Metode:
        GUI(catalog, engine) — constructor; pastreaza referinte, cursorPos=0.
        run() — InitWindow, loop: handleInput(), BeginDrawing(), drawInputPanel/drawResultsPanel/drawLogPanel(), EndDrawing(), CloseWindow().
    Private:
        handleInput() — GetCharPressed (insereaza la cursorPos), KEY_LEFT/KEY_RIGHT (miscare cursor), KEY_BACKSPACE (stergere + repeat la hold), KEY_ENTER (executeQuery).
        executeQuery() — log "> input", engine.query(input), log "[OK]..." sau "[ERROR]...", results = ..., input.clear(), cursorPos=0.
        drawInputPanel() — fundal, dreptunghi input, text input, cursor clipitor la cursorPos, buton RUN (hover + click = executeQuery).
        drawResultsPanel() — header "RESULTS", randurile din results (alternand culori).
        drawLogPanel() — header "LOG", buton DARK/LIGHT (toggle isDark), liniile din logs (verde/rosu pentru ERROR).

StringUtils/
----------------------------------------------------------------

split(str, delimiter)
    Rol: Sparge un string dupa un caracter; returneaza vector<string>.
    Folosit in Table pentru deserializare "v1|v2|v3" -> { "v1", "v2", "v3" }.
